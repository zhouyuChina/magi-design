## 1. 定位与范围

### 1.1 定位

为业务线运营人员提供可视化的 SOP 管理界面，替代直接使用 Swagger UI 录入的人工操作，降低操作门槛并减少录入错误。

### 1.2 交付范围

| 功能 | 说明 |
|------|------|
| SOP 列表页 | 展示所有 SOP，支持状态筛选、关键字搜索 |
| 新建 SOP | 表单填写 + 动态步骤编辑器 |
| 编辑 SOP | 与新建复用同一表单，intent_key 只读 |
| 启用 / 禁用 SOP | 列表行内操作 |
| 删除 SOP | 软删除，二次确认 |

---

## 2. 技术选型

| 层级 | 方案 | 说明 |
|------|------|------|
| 框架 | React 18 + TypeScript | 类型安全，减少运行时错误 |
| UI 组件库 | Ant Design 5 | 开箱即用的企业级组件（表格、表单、Switch 等） |
| 构建工具 | Vite 5 | 开发启动快，生产构建快 |
| 服务端状态 | TanStack Query v5 | 自动缓存 + 失效，减少手写 loading/error 逻辑 |
| 路由 | React Router v6 | 标准 SPA 路由 |
| HTTP 客户端 | Axios | 统一拦截器处理错误提示 |
| 代码规范 | ESLint + Prettier | 与后端同步 Conventional Commits 规范 |

```
├──────────────────────────────┬──────────────────────────────┤

├──────────┬────────┬──────┬───┴────────┬──────┬─────────────┤
│ 意图标识 │ 显示名 │全局  │可中断      │步骤数│ 状态 │ 操作 │
├──────────┼────────┼──────┼────────────┼──────┼──────┼──────┤
│ verify.. │ 身份.. │  ✗   │  ✗         │  2   │ 启用 │编辑  │
│          │        │      │            │      │      │禁用  │
│          │        │      │            │      │      │删除  │
├──────────┼────────┼──────┼────────────┼──────┼──────┼──────┤
│ query_p..│ 优惠.. │  ✓   │  ✓         │  1   │ 启用 │编辑  │
│          │        │      │            │      │      │禁用  │
│          │        │      │            │      │      │删除  │
└──────────┴────────┴──────┴────────────┴──────┴──────┴──────┘
```

**表格列说明**

| 列 | 绑定字段 | 备注 |
|----|---------|------|
| 意图标识 | `intent_key` | 等宽字体展示，点击可复制 |
| 显示名 | `intent_label` | |
| 全局意图 | `is_global` | Tag：全局 / 流程 |
| 可中断 | `allow_interrupt` | Tag |
| 步骤数 | `steps_config.length` | |
| 状态 | `status` | Badge：启用（绿）/ 禁用（灰） |
| 操作 | — | 编辑、启用/禁用切换、删除 |

**交互说明**

- 搜索框实时过滤 `intent_key` 和 `intent_label`（前端过滤，无需额外请求）
- 状态筛选传 `?status=` Query 参数重新请求后端
- 启用/禁用：直接 PUT 更新 `status` 字段，无需跳转页面
- 删除：弹出 Ant Design `Modal.confirm` 二次确认后执行 DELETE

**字段说明**

| 字段 | 组件 | 编辑时行为 |
|------|------|-----------|

| `steps_config` | StepsEditor（自定义） | 见 5.3 节 |
| `active_intents` | Select（mode="tags"） | 自由输入字符串 tag |

## 5. 组件设计

### 5.1 `<SOPTable />`

- 使用 Ant Design `Table`，数据来自 `useSOPList()` hook
- 操作列使用 `Popconfirm` 包裹删除按钮，避免误操作

```typescript
// Props
interface SOPTableProps {
  data: SOPDefinition[];
  loading: boolean;
  onEdit: (intentKey: string) => void;
  onToggleStatus: (intentKey: string, currentStatus: number) => void;
  onDelete: (intentKey: string) => void;
}
```

### 5.2 `<SOPForm />`

- 使用 Ant Design `Form`，通过 `initialValues` 区分新建与编辑
- 提交时触发 `onSubmit(values)` 回调，由父页面决定调用 create 或 update API

```typescript
// Props
interface SOPFormProps {
  initialValues?: Partial<SOPDefinition>;
  isEdit?: boolean;      // true 时 intent_key 为只读
  onSubmit: (values: SOPFormValues) => Promise<void>;
  onCancel: () => void;
}
```

### 5.3 `<StepsEditor />`

`steps_config` 的动态编辑器，核心逻辑：

- 用 React state 维护步骤数组，`step` 索引由位置自动计算（不手动输入），保证连续性
- 支持添加步骤（末尾追加）、删除步骤（重新计算索引）
- 每行包含：`description`（Input）、`required_slots`（Select tags 模式）、`api_endpoint`（Input，留空时提交为 null）

```typescript
// Props
interface StepsEditorProps {
  value?: StepConfig[];
  onChange?: (steps: StepConfig[]) => void;  // 兼容 Ant Design Form.Item 受控模式
}
```

### 5.4 API 服务层 `services/sop.ts`

```typescript
// All endpoints map to backend API v1
const sopApi = {
  list: (status?: number) =>
    axios.get<SOPListResponse>('/api/v1/admin/sop', { params: { status } }),

  get: (intentKey: string) =>
    axios.get<SOPDefinition>(`/api/v1/admin/sop/${intentKey}`),

  create: (data: SOPCreatePayload) =>
    axios.post<{ intent_key: string; status: number }>('/api/v1/admin/sop', data),

  update: (intentKey: string, data: SOPUpdatePayload) =>
    axios.put<{ intent_key: string; updated_at: string }>(
      `/api/v1/admin/sop/${intentKey}`, data
    ),

  remove: (intentKey: string) =>
    axios.delete<{ intent_key: string; status: number }>(
      `/api/v1/admin/sop/${intentKey}`
    ),
};
```

### 5.5 TanStack Query Hooks

```typescript
// hooks/useSOP.ts

// Read
export const useSOPList = (status?: number) =>
  useQuery({ queryKey: ['sop', status], queryFn: () => sopApi.list(status) });

export const useSOPDetail = (intentKey: string) =>
  useQuery({ queryKey: ['sop', intentKey], queryFn: () => sopApi.get(intentKey) });

// Mutations（成功后自动 invalidate 列表缓存）
export const useCreateSOP = () =>
  useMutation({ mutationFn: sopApi.create,
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['sop'] }) });

export const useUpdateSOP = () =>
  useMutation({ mutationFn: ({ key, data }) => sopApi.update(key, data),
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['sop'] }) });

export const useDeleteSOP = () =>
  useMutation({ mutationFn: sopApi.remove,
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['sop'] }) });
```

## 6. 表单校验

与后端校验规则保持一致，在前端提前拦截明显的输入错误：

| 字段 | 规则 | 错误提示 |
|------|------|---------|
| `intent_key` | 必填；正则 `/^[a-z0-9_]{2,64}$/` | 只允许小写字母、数字、下划线，长度 2-64 |
| `intent_label` | 必填；长度 1-128 | 请填写显示名 |
| `steps_config` | 至少 1 个步骤；每步 `description` 必填 | 至少配置 1 个步骤 / 步骤描述不能为空 |
| `api_endpoint` | 非空时必须以 `http://` 或 `https://` 开头 | 请输入合法的 URL |

> **注意**：`step` 索引由 `StepsEditor` 内部维护，前端不暴露给用户填写，从根本上避免索引不连续的错误。

## 7. 错误处理

Axios 响应拦截器统一处理：

| HTTP 状态 | 行为 |
|-----------|------|
| `409 Conflict` | Ant Design `message.error('意图标识已存在，请更换')` |
| `422 Unprocessable Entity` | 展示后端返回的 `error` 字段内容 |
| `404 Not Found` | `message.error('SOP 不存在，可能已被删除')` + 跳回列表页 |
| `5xx` | `message.error('服务异常，请稍后重试')` |

## 9. 类型定义

```typescript
// types/sop.ts

export interface StepConfig {
  step: number;
  description: string;
  required_slots: string[];
  api_endpoint: string | null;
}

export interface SOPDefinition {
  intent_key: string;
  intent_label: string;
  allow_interrupt: boolean;
  is_global: boolean;
  steps_config: StepConfig[];
  active_intents: string[];
  status: 0 | 1;
  created_by?: string;
  created_at: string;
  updated_at: string;
}

export type SOPCreatePayload = Omit<SOPDefinition, 'status' | 'created_at' | 'updated_at'>;
export type SOPUpdatePayload = Omit<SOPCreatePayload, 'intent_key'>;

export interface SOPListResponse {
  data: SOPDefinition[];
  total: number;
}
```

## 11. 边界说明

- **认证鉴权**：本期不做登录，假设内网部署；后续如需接入 SSO 再扩展
- **国际化**：界面仅支持中文，不做 i18n 框架封装
- **步骤排序**：本期不做拖拽排序，只支持末尾追加和任意位置删除
- **移动端**：不做响应式适配，仅面向桌面浏览器
